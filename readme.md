# How to read and run
The Main.class asks for input in form of any text for <b>until we provide two new lines</b>. After two new lines were provided another question is asked for desired output format. Provide answer, either XML or CSV, and output in given format will be printed to standard output.

Code supports reading from Std::in and Resource files.

# How to use tests
There are two MAIN test files:
- src/test/java/org/amen/MainSmallTest.java
- src/test/java/org/amen/MainLargeTest.java

First one passes for input provided in the sample. Second test file requires:
- sample-files/large.xml
- sample-files/large.csv.

You can <b>replace resource files with Your own</b> valid files and run the test file. It should assert if there are any differences between output generated by the code and files found in resources.

# Missing features
If there are files in text with extensions they are not correctly interpreted as break of sentence. Files and their extensions should be kept together and properly interpreted as part of sentence.
Example sentence which won't be processed properly:
`File MainSmallTest.java should not be broken into parts.`
